# wrap-up: 이어서 하자

오늘 한 걸 갈무리하고 내일의 나에게 쪽지를 남기는 일. 여행 다녀와 짐 풀고 영수증 붙여두는 것과 같다. 안 해두면 다음 세션의 나는 백지에서 시작한다.

**철칙(Iron Law): 기록 없이 끝내지 않는다. 내일의 나는 오늘을 기억 못 한다.** 인수인계 3칸(끝낸 것/다음 것/막힌 것)과 내일 첫 마디를 남기기 전에는 세션을 닫지 않는다.

## 절차

### 1. 오늘 한 걸 확인해 기록에 반영
"오늘 뭘 했는지 먼저 맞춰봐요." 라고 말하고 진행한다.
- plan.md 가 있으면 오늘 끝난 항목의 체크박스를 갱신한다.
- plan.md 가 없으면 강요하지 말고, "오늘 한 일 3줄 요약 파일을 하나 남겨둘까요?" 하고 제안만 한다.

### 2. 인수인계 3칸
"내일의 나한테 남기는 쪽지예요. 딱 세 줄." 이라고 말한다.
- 끝낸 것 / 다음 것 / 막힌 것 을 각각 한 줄로 정리해 보여준다. 막힌 게 없으면 "막힌 것: 없음" 이라 적는다.

### 3. 저장하고 마무리 (git 있으면)
"오늘 것 안 날아가게 저장해둘게요." 라고 말한다.
- git 저장소면 변경을 커밋하고 원격에 push 한다. 없으면 이 단계는 조용히 건너뛴다.

### 4. 내일 첫 마디
"내일 이 한 줄만 저한테 던지면 바로 이어서 가요." 하고, 다음 세션에 그대로 붙여넣을 수 있는 시작 문장 한 줄을 만들어 준다.

## 금지
- 기록·인수인계 없이 그냥 "끝" 하는 것.
- plan.md 없는데 파일을 강제로 만드는 것 (제안까지만).
- 확인 없이 커밋 메시지에 오늘 안 한 일까지 지어 넣는 것.
- .env 값을 기록에 적는 것 (키 이름만).

## 막히면
정리할 게 너무 많아 뒤엉키면 "가장 마지막에 만진 것 하나"만 먼저 적고 나머지는 다음 세션으로 미룬다. 그래도 뭘 남길지 모르겠으면 강사·커뮤니티에 물어라.

## 졸업 한 줄
이 스킬 없이도 기억할 세 마디: 끝낸 것 → 다음 것 → 막힌 것.
