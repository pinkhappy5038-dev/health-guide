# tidy: 방 정리 스킬

한 줄 정의: **동작은 그대로 두고 방만 치우는 것.** 이사가 아니라 청소다. 가구를 옮기고 안 쓰는 물건을 버려도, 그 방에서 살던 사람의 생활은 똑같아야 한다.

## 철칙 (Iron Law)

**동작은 1도 바뀌지 않는다.** 청소 전과 후에 화면과 기능이 완전히 같아야 청소다. 조금이라도 달라졌다면 그건 청소가 아니라 사고이므로 즉시 되돌린다.

## 절차

### 1. 청소 전 안전망: 지금 멀쩡한가부터 확인
- 사용자에게: "치우기 전에 지금 화면이 제대로 도는지부터 봅니다. 고장 난 방은 못 치워요."
- package.json 을 읽어 실행 명령을 찾고, 실제 화면을 열어 눌러본다(자동 테스트가 있으면 같이 돌린다).
- **안 돌아가면 여기서 멈춘다.** "지금 버그가 있어요. 정리 말고 이것부터 잡죠"라고 말하고 종료. 망가진 걸 치우면 뭐가 청소 때문인지 알 수 없다.

### 2. 버릴 것·합칠 것 목록만 먼저 (승인 게이트)
- 사용자에게: "뭘 버리고 뭘 합칠지 목록만 먼저 보여드릴게요. 손대기 전에 골라주세요."
- 안 쓰는 것 / 중복된 것 / 너무 커진 파일을 찾아 표로 제시한다(파일·이유·처리방식). **아직 아무것도 건드리지 않는다.**
- 사용자가 고른 항목만 진행한다. 여러 파일을 갈아엎기 전 반드시 이 승인을 받는다.

### 3. 한 번에 한 종류만: 버리기, 합치기, 나누기
- 사용자에게: "이번엔 안 쓰는 것만 지웁니다. 다른 건 안 건드려요."
- 순서 고정: (1) 안 쓰는 코드·파일 삭제 → (2) 중복 합치기 → (3) 큰 파일 나누기. 세 가지를 섞지 않는다.
- 파괴적 삭제 전에는 사용자 확인을 받는다.

### 4. 종류마다 "아직 되나?" 재확인
- 사용자에게: "한 종류 끝났으니 다시 열어서 눌러봅니다. 아까랑 똑같은지 보는 거예요."
- 1단계와 같은 방법으로 화면을 다시 확인한다. 같으면 다음 종류로.

### 5. 다르면 즉시 되돌리기
- 화면·기능이 조금이라도 달라졌으면 사용자에게: "달라졌어요. 방금 것만 되돌립니다." 하고 그 변경만 원복한다.
- 원인을 찾을 때까지 더 나아가지 않는다. 철칙이 깨진 상태로는 한 발도 못 간다.

## 금지
- 동작을 바꾸는 것(기능 추가·수정·"개선"). 이건 청소가 아니다.
- 목록 승인 없이 여러 파일을 한꺼번에 갈아엎는 것.
- 안 쓰는 것 지우기·합치기·나누기를 한 턴에 섞는 것.
- 화면 확인 없이 다음 종류로 넘어가는 것.
- .env 를 읽거나 출력하는 것(키 이름만 다룬다).

## 막히면
더 작게 쪼갠다. 파일 하나가 안 되면 그 파일 안에서 함수 하나만 손보고 다시 확인한다. 그래도 계속 어긋나면 멈추고 사람(강사·커뮤니티)에게 지금까지 한 것과 어디서 달라졌는지 보여준다.

## 졸업 한 줄
이 스킬 없이도 기억할 세 마디: **되는지 먼저 확인 → 한 종류씩 → 달라지면 되돌린다.**
